**1. Extend `get_game_state` in `app/services.py`:**

Add an `actions` field to the `GameState` dataclass containing all ScoreActions for the game (both active and undone), each with their associated ScoreEntries, ordered by `id ASC` (chronological). The template needs: action.id, action.event_type, action.description, action.is_undone, and the list of entries (each with player_id and points).

Load actions with a query like:
```python
actions = session.exec(
    select(ScoreAction)
    .where(ScoreAction.game_id == game_id)
    .order_by(ScoreAction.id)
).all()
```

For each action, load its entries:
```python
for action in actions:
    entries = session.exec(
        select(ScoreEntry).where(ScoreEntry.action_id == action.id)
    ).all()
```

Create a simple dataclass `ActionWithEntries` to hold each action plus its entries and player names/colors. The template needs player name and color for each entry, so either:
- Preload players into a dict `{player_id: Player}` and pass it to the template, OR
- Build a list of dicts/dataclasses with player info baked in.

The simpler approach: add `actions: list` to GameState, where each item is a dataclass:
```python
@dataclass
class ActionDetail:
    action: ScoreAction
    entries: list[dict]  # Each dict: {"player_name": str, "player_color": str, "points": int}
```

Add `action_details: list[ActionDetail]` to `GameState`.

**3. Add the `history` block to `app/templates/dashboard.html`:**

Place it after the `action_bar` block, before `</div>` (end of container). Use a `{% block history %}` named block so it can be rendered as an OOB fragment.

Structure (based on the prototype at `docs/prototype/dashboard.html`):
```html
{% block history %}
<div id="history" class="history"
     {% if oob %}hx-swap-oob="true"{% endif %}>
    <div class="history-header">
        <span class="history-title">Historial de acciones</span>
    </div>
    <ul class="history-list">
        {% for detail in action_details|reverse %}
        <li class="history-item {% if detail.action.is_undone %}undone{% endif %}">
            <span class="history-number">#{{ detail.action.id }}</span>
            <div class="history-content">
                <span class="history-type">{{ EVENT_TYPE_LABELS[detail.action.event_type] }}: </span>
                <span class="history-players">
                    {% for entry in detail.entries %}
                    <span style="color:{{ PLAYER_COLORS[entry.player_color].hex }}">{{ entry.player_name }}</span> +{{ entry.points }}{% if not loop.last %}, {% endif %}
                    {% endfor %}
                </span>
                {% if detail.action.description %}
                <div class="history-note">{{ detail.action.description }}</div>
                {% endif %}
            </div>
            {% if not detail.action.is_undone %}
            <button class="history-rollback-btn"
                    hx-post="/games/{{ game.id }}/rollback"
                    hx-vals='{"action_id": "{{ detail.action.id }}"}'
                    hx-target="#score-table"
                    hx-swap="outerHTML"
                    hx-confirm="Rollback to action #{{ detail.action.id }}?">
                &#8630;
            </button>
            {% endif %}
        </li>
        {% else %}
        <li class="history-item" style="color:var(--text-muted); justify-content:center">
            Sin acciones todavia
        </li>
        {% endfor %}
    </ul>
</div>
{% endblock %}
```

**Important:** The `|reverse` filter shows newest first (matches prototype). The rollback button appears only on active (non-undone) actions. The `hx-confirm` provides a safety prompt before rollback.

**4. Add the rollback route to `app/web/routes.py`:**

```python
@router.post("/games/{game_id}/rollback")
def rollback_action(
    request: Request,
    game_id: int,
    action_id: int = Form(...),
    session: Session = Depends(get_session),
):
    """Rollback to a specific action and return HTMX fragments."""
    try:
        rollback_to(session, game_id, action_id)
    except ValueError:
        pass
    return _render_dashboard_fragments(request, session, game_id)
```

Add `rollback_to` to the imports from `app.services`.
