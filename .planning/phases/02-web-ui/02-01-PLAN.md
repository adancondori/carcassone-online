  <action>
    1. Add setup service functions to app/services.py (if not already present from Phase 1):
       - create_game(session, name: str) -> Game: creates Game with name, status="setup", commits, returns game
       - add_player(session, game_id: int, name: str, color: str) -> Player:
         - Load game, verify status == "setup"
         - Count existing players, raise ValueError if >= 6
         - Determine turn_order = count + 1
         - Create Player, commit, return
       - remove_player(session, game_id: int, player_id: int) -> None:
         - Load game, verify status == "setup"
         - Delete player, reorder remaining players' turn_order sequentially, commit
       - start_game(session, game_id: int) -> Game:
         - Load game, verify status == "setup"
         - Count players, raise ValueError if < 2
         - Set game.status = "playing", commit, return game

       NOTE: Phase 1 plan 01-02 (services) only defines add_score, undo_last, rollback_to, recalculate_score. It does NOT define create_game, add_player, remove_player, start_game. These MUST be added in this plan.

    2. Create app/web/routes.py:
       - Import APIRouter, Request, Form, Depends from fastapi
       - Import RedirectResponse from fastapi.responses
       - Import Session from sqlmodel
       - Import templates, PLAYER_COLORS from app.web.dependencies
       - Import get_session from app.db
       - Import service functions from app.services
       - Create router = APIRouter()

       Routes (ALL use `def`, NOT `async def` -- sync handlers for sync DB):

       GET /games/new:
         - Render setup.html with {"colors": PLAYER_COLORS, "game": None, "players": [], "error": None}
         - This is the initial page with just the game name form

       POST /games:
         - Form params: name: str = Form(...)
         - Call create_game(session, name)
         - RedirectResponse(url=f"/games/{game.id}/setup", status_code=303)

       GET /games/{game_id}/setup:
         - Load game_state via get_game_state(session, game_id)
         - If game.status != "setup", redirect to /games/{game_id} (dashboard)
         - Compute available_colors = {k: v for k, v in PLAYER_COLORS.items() if k not in [p.color for p in players]}
         - Render setup.html with {"game": game, "players": players, "colors": PLAYER_COLORS, "available_colors": available_colors, "error": None}

       POST /games/{game_id}/players:
         - Form params: name: str = Form(...), color: str = Form(...)
         - Try: add_player(session, game_id, name, color)
         - Except ValueError as e: redirect back with error flash (or just redirect; errors will be handled by constraint violations)
         - RedirectResponse(url=f"/games/{game_id}/setup", status_code=303)

       POST /games/{game_id}/players/{player_id}/delete:
         - Call remove_player(session, game_id, player_id)
         - RedirectResponse(url=f"/games/{game_id}/setup", status_code=303)

       POST /games/{game_id}/start:
         - Call start_game(session, game_id)
         - RedirectResponse(url=f"/games/{game_id}", status_code=303)

       GET /games/{game_id}:
         - Load game_state
         - If game.status == "setup", redirect to /games/{game_id}/setup
         - Render dashboard.html (stub for now -- just render a placeholder that says "Dashboard coming in 02-02")
         - Pass: game, players, PLAYER_COLORS, has_actions=(action_count > 0)
