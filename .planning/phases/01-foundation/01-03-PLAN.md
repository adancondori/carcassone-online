  <files>
    alembic.ini

    alembic/versions/

  <action>
    Run `alembic init alembic` from the project root to scaffold the alembic directory.

    Edit alembic.ini:
    - Set sqlalchemy.url = sqlite:///data/carcassonne.db

    Edit alembic/env.py:
    - Add imports: `from sqlmodel import SQLModel`, `from app.models import *` (import ALL models so metadata is populated), `from app.config import settings`
    - Set `target_metadata = SQLModel.metadata`
    - In run_migrations_online(), configure context with:
      - `render_as_batch=True` (CRITICAL for SQLite)
      - `user_module_prefix="sqlmodel.sql.sqltypes."` (for SQLModel custom types)
    - In run_migrations_offline(), also add render_as_batch=True and user_module_prefix

    Edit alembic/script.py.mako:
    - Add `import sqlmodel.sql.sqltypes` after the existing imports block

    Generate the initial migration:
    - `alembic revision --autogenerate -m "initial schema"`
    - Verify the generated migration file creates all 4 tables (game, player, score_action, score_entry) with constraints

    Run the migration:
    - Create data/ directory if it doesn't exist: `mkdir -p data`
    - `alembic upgrade head`
    - Verify with: `python -c "import sqlite3; conn = sqlite3.connect('data/carcassonne.db'); print([r[1] for r in conn.execute('SELECT * FROM sqlite_master WHERE type=\"table\"').fetchall()])"` -- should show all 4 tables
  </action>
  <verify>
    alembic upgrade head (runs without error)
    python -c "import sqlite3; conn = sqlite3.connect('data/carcassonne.db'); tables = [r[1] for r in conn.execute(\"SELECT * FROM sqlite_master WHERE type='table'\").fetchall()]; assert 'game' in tables and 'player' in tables and 'score_action' in tables and 'score_entry' in tables; print('All tables exist:', tables)"
  </verify>

<task type="auto">
  <name>Task 2: Create FastAPI app shell and Docker configuration</name>
  <files>
    app/main.py
    Dockerfile
    docker-compose.yml
  </files>
  <action>
    Create app/main.py:
    - Import asynccontextmanager from contextlib
    - Import FastAPI
    - Import create_db_and_tables, engine from app.db
    - Define lifespan async context manager:
      - On startup: create data/ directory (os.makedirs("data", exist_ok=True)), call create_db_and_tables()
      - On shutdown: engine.dispose()
    - Create FastAPI app with lifespan=lifespan, title="Carcassonne Scoreboard"
    - Add a health check endpoint: GET / returning {"status": "ok"} (useful for verifying Docker works)

    Create Dockerfile:
    - FROM python:3.12-slim
    - WORKDIR /code
    - ENV PYTHONDONTWRITEBYTECODE=1, PYTHONUNBUFFERED=1
    - COPY pyproject.toml .
    - RUN pip install --upgrade pip && pip install -e ".[dev]"
    - COPY . .
    - RUN mkdir -p /code/data
    - EXPOSE 8000
    - CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]

    Create docker-compose.yml:
    - services.web:
      - build: .
      - container_name: carcassonne_web
      - command: uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
      - ports: "8000:8000"
      - volumes: .:/code (bind mount for reload), db_data:/code/data (named volume for data directory)
      - env_file: .env
    - volumes: db_data: (named volume persists across down/up)

    CRITICAL: The volume mounts /code/data (DIRECTORY), NOT /code/data/carcassonne.db (file). This ensures .db-wal and .db-shm files are also persisted.

    Add .dockerignore with: .git, __pycache__, *.pyc, .env (optional -- docker-compose passes env_file separately), data/ (don't copy local data into image)

    Verify locally (without Docker): `uvicorn app.main:app --port 8000 &` then `curl http://localhost:8000/` returns {"status":"ok"}, then kill the process.
  </action>
  <verify>
    docker compose up --build -d && sleep 3 && curl -s http://localhost:8000/ && docker compose down
    Expected: {"status":"ok"} from the health endpoint
  </verify>
  <done>
    FastAPI app starts with lifespan. Docker builds and runs. Health check responds. Named volume persists data directory across restarts. All INFRA requirements (01, 02, 03, 05) are satisfied.
  </done>
</task>
